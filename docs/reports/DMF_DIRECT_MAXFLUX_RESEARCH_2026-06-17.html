Direct MaxFlux 方法与 ATST-Tools 集成可行性调研

Version: 2026-06-17 Updated evidence: 2026-06-19 Branch: feat/dmf Scope: research report DMF source: temp_repos/dmf @ ed7ed53

结论摘要。Direct MaxFlux（DMF）是值得 ATST-Tools 跟进的双端过渡态候选结构生成方法，但不应立即替代当前生产级 NEB/CI-NEB/AutoNEB 工作流。短期更合理的定位是：先作为可选依赖的研究性 double-ended TS guess workflow 建立非周期基线，再尽早验证 ATST 核心周期体系风险；D2S 集成应放在 PBC 语义和生产级案例证据稳定之后。其输出最高能点仍需接 Dimer/Sella/CCQN、频率分析和 IRC 验证。

周期性体系判断。当前 PyDMF 与原始论文口径不能被视为严格支持周期性体系。论文摘要页明确说明当前只支持非周期体系；源码虽通过 ASE Atoms.copy() 保留 cell/PBC 元数据，但路径变量、FB-ENM/CFB-ENM 距离、速度约束和旋转/平动处理均基于未显式 minimum-image 的 Cartesian positions。固定晶胞、端点已 unwrap 到同一周期图像、且路径不跨 cell 分支的 slab/扩散案例可以实验性尝试；锂离子跨周期边界迁移、表面吸附解离、晶胞变化或需要 fractional-coordinate 连续性的案例，必须先补 ATST 侧 PBC 适配和验证。

方法定位

DMF 是 double-ended variational path optimization，不是最终一阶鞍点优化器。

工程价值

以少量 evaluation points 优化 B-spline 路径，适合昂贵 DFT/MLP 后端生成 TS 初猜。

ATST 建议

按“非周期基线 -> PBC 实验 -> 生产案例 -> D2S 支持”推进；周期语义先于高层编排固化。

0. 当前实现与验证状态（2026-06-21）

当前状态。DMF 已在 ATST-Tools 中完成实验性集成，但仍不是 NEB/CI-NEB/AutoNEB 的生产级替代。当前定位是 experimental TS candidate / rough path generator：可独立运行 calculation.type: dmf，也可在 D2S 中作为 opt-in rough_method: dmf 接 Sella、局域振动、descent-IRC 和 ABACUS 单点对照。2026-06-21 修复后，ATST 会把 YAML nmove 传给最终 DirectMaxFlux，summary 记录最终 t_eval；D2S rough-DMF 也按实际 t_eval 选择 tmax 邻域。

0.3 计算效率：是否快于传统 CI-NEB？

短答：在已有两例验证中，DMF 作为 candidate/rough-stage generator 的 walltime 明显短于传统 ABACUS two-stage NEB/CI-NEB；但这不是严格 apples-to-apples 的“同后端完整 TS workflow”比较，因为 DMF 主要用 DP 生成候选，再用 Sella/validation/ABACUS comparison 补证据，而 CI-NEB 是 ABACUS 路径优化本体。

0.5 方法特点与边界

  • 特点：少量 B-spline/evaluation points 生成 TS candidate，适合昂贵后端前的 MLP/DP 预筛选和 D2S rough stage。
  • 输出语义：DMF 输出的是 tmax candidate，不是严格 saddle；必须接 Sella/Dimer/CCQN、振动和 IRC/endpoint validation。
  • PBC 边界：当前只承认固定 cell、端点已预 unwrap 到同一周期图像、Cartesian positions 连续的实验路径；不支持自动 MIC、fractional-coordinate path、跨边界迁移自动识别或 variable-cell path。
  • 输入边界：必须保证 atom order/mapping、constraints、端点图像和固定原子位置一致；失败案例显示这些不一致会导致 IPOPT 长时间迭代且无产物。
  • 支持边界：DMF 和 DMF-D2S 保持 experimental；成功案例说明“值得作为 TS guess accelerator 使用”，失败案例说明“不能直接替代 CI-NEB 的稳健生产路径”。

2. DMF 算法定位

DMF 的核心不是从单个结构沿负曲率方向收敛到鞍点，而是给定反应物和产物两端结构后，优化一条连续反应路径。路径最高能点在 large-β 极限下成为 transition-state candidate，再进入精确 TS optimization、虚频分析和 IRC 验证。

PyDMF 文档和源码将路径写为 B-spline 展开，优化变量是内部 B-spline 控制点。其 Direct MaxFlux 泛函为：

I[x] = integral_0^1 |dx/dt| exp(beta E(x(t))) dt
objective = beta^-1 log I[x]

当 beta 较大时，指数项强调路径上的高能区域；优化该泛函等价于压低路径最大能量，同时速度约束保持路径参数化合理。PyDMF 实现中 DirectMaxFlux._get_objective() 返回 log(action) / beta，_get_func_en() 返回 exp(beta * en) 及其导数。

2.1 典型工作流

  1. 准备 reactant/product 两端结构，要求原子顺序、映射、电荷/自旋和构象选择合理。
  2. 用 FB-ENM 或 CFB-ENM 生成 chemically plausible initial path，避免线性插值碰撞。
  3. 把初始 B-spline coefficients 传入 DirectMaxFlux，配置 ASE calculators。
  4. 用 IPOPT/cyipopt 求解非线性约束优化问题。
  5. 通过 energy interpolation 找到 tmax，输出路径最高能结构。
  6. 用 Dimer、Sella、CCQN 或后端原生 TS optimizer 做最终鞍点优化和验证。

2.2 DMF 与 MaxFlux 传统来源

DMF 源自 MaxFlux reaction path 思想，但 PyDMF 中面向实际 TS 搜索的实现已转化为一个低温极限下的变分路径优化问题。相比传统 MaxFlux 的有限温度扩散通量解释，当前 PyDMF 的工程目标更明确：用较少能量/力评估点定位靠近 TS 的路径最高能结构。

3. DMF 与 CI-NEB 的比较

维度 CI-NEB / NEB DMF / PyDMF 对 ATST-Tools 的含义
路径表示 离散 images 加弹簧和投影力；CI image 沿路径上坡、垂直方向下降。 B-spline 连续路径；优化内部控制点和少量 evaluation points。 DMF 更像曲线优化器，不能直接复用 ATST 当前 NEB trajectory 语义。
目标函数 NEB 力投影规则，不是普通标量目标函数最小化。 显式 MaxFlux 泛函，交给 IPOPT 做非线性约束优化。 需要引入 cyipopt/IPOPT 依赖和 optimizer diagnostics。
计算点数量 通常需要较多 images 保证路径分辨率；CI-NEB 生产经验成熟。 默认 nmove=5，总评估点 nmove + 2，可自适应集中到高能区。 对 ABACUS/DFT 后端有潜在节省，但收益需用 ATST 案例实测。
最终 TS 可靠性 CI image 可接近 saddle，但仍常需 Dimer/Sella/frequency/IRC 验证。 最高能点是 TS candidate，不是严格鞍点收敛结果。 应接入现有 D2S/TS validation，而不是单独给出最终结论。
PBC 生产经验 ASE NEB 与 ATST 当前 Li-Si、H2-Au、Cy-Pt、Zn migration 案例已形成验证链。 公开论文口径和源码实现均未给出严格 PBC 支持。 周期体系不能直接宣传为 supported，必须先做受控实验性功能。

因此，DMF 对 ATST 的价值不是“替换 CI-NEB”，而是先作为独立 TS-guess accelerator 验证其对 ATST 核心案例的适用性；只有当 PBC 语义和生产级对照案例稳定后，才适合补充当前 D2S 工作流，用更少路径点生成 TS 初猜并进入 ATST 已有单端 TS refinement 与验证链。

4. PyDMF 当前源码实现

4.1 包结构与依赖

PyDMF 当前 pyproject.toml 显示包名为 pydmf，版本 1.2.1，基础依赖为 ase>=3.22；cyipopt 是 optional dependency，但 src/dmf/dmf.py 顶层直接 import cyipopt，实际使用 DirectMaxFlux 时需要可用 IPOPT/cyipopt 环境。官方 README 推荐 conda 安装，这是 ATST 集成时必须保留为 optional extra 的原因。

模块 职责 关键观察
src/dmf/dmf.py VariationalPathOpt 和 DirectMaxFlux。 B-spline basis、速度/平动/转动约束、energy/force evaluation、IPOPT callbacks、tmax 历史记录。
src/dmf/interpolate.py interpolate_fbenm()。 先用 FB-ENM/CFB-ENM 做路径生成，再返回可复用的 coefficients。
src/dmf/fbenm.py FB-ENM、FB_ENM_Bonds、CFB_ENM calculators。 基于 pair distances、covalent/vdW radii、bond graph、quartets 和 planar fragments 构造路径约束。
src/dmf/torch/* PyTorch/GPU 版本。 API 与 NumPy 版本相近，但依赖 torch；可作为未来加速方向，不建议首轮接入。
tests/test_dmf.py 基础 import、FB-ENM coefficients、EMT DirectMaxFlux smoke。 测试覆盖非周期小分子；没有 PBC/ABACUS/DP calculator 验证。

4.2 路径与优化变量

VariationalPathOpt.__init__ 以 ref_images[0].copy() 生成 self.images，因此 cell/PBC 元数据会随 ASE Atoms 被复制。但优化变量是 coefs[1:-1] 的 Cartesian coordinates；端点 coefficients 固定，若启用 remove_rotation_and_translation，还会把最终端点旋转角作为优化变量。

这套设计对分子体系很合理：它通过 mass center 平移和逐帧旋转对齐移除刚体冗余。但对周期 slab 或晶体，默认旋转/平动冗余处理并不自然：晶胞方向本身有物理意义，表面法向和吸附位点不应被全局旋转消除。

4.3 能量/力评估与并行

_get_forces_by_img_idxs() 支持 serial 或 Python thread-based parallel evaluation；README_PARALLEL 明确要求用户为每个 image 分配独立 calculator 资源和输出目录。ATST 当前 NEB/AutoNEB image-level MPI 已有“每个 image 独立 calculator directory”的治理经验，这一点可复用；但 PyDMF 的并行粒度是线程并发，不等同于 ATST 当前 MPI world 分配。

4.4 FB-ENM / CFB-ENM 路径生成

FB_ENM 和 CFB_ENM 是 ASE calculators，用 flat-bottom pair potentials 构造无碰撞且保留非反应结构的路径。FB_ENM_Bonds 基于 covalent radii 识别 bonds，并给强约束和弱约束设置 d_min/d_max；CFB_ENM 进一步识别断键/成键 pair quartets，调控成键断键时机。

这一实现对有机/分子反应特别有价值，但对周期体系需要谨慎：源码中距离主要来自 atoms.get_positions()、手写 Cartesian pair differences 或默认 atoms.get_all_distances()。报告期未发现 mic=True、scaled coordinates、wrap/unwrap 或 fractional displacement 的显式处理。

5. 周期性体系适用性

核心判断：DMF 的思想可以被扩展到周期体系，但当前 PyDMF 实现和原始论文口径不构成严格周期体系支持。对 ATST-Tools 来说，周期体系 DMF 必须先作为实验性功能处理。

5.1 外部证据

Koda 与 Saito 2024 年 JCTC 论文 DOI 页面摘要/说明中明确指出当前仅支持非周期体系。该表述与本地 PyDMF 源码观察一致：虽然文档字符串说 ref_images[0] 会保留 cell 和 PBC settings，但核心几何操作仍在 Cartesian positions 上进行。

5.2 源码层面的 PBC 风险

问题 源码证据 周期案例风险
路径坐标不是 fractional coordinates get_positions()、set_positions()、B-spline coefficients 全部基于 Cartesian positions。 离子跨边界迁移会出现走长路或穿越错误周期镜像的问题。
未显式 minimum-image 距离 FB-ENM/CFB-ENM 使用 get_positions() 手写 pair differences，或默认 get_all_distances()。 表面 adsorbate 与 slab、扩散离子与邻近晶胞原子的约束可能错误。
默认旋转/平动冗余处理适合孤立分子 remove_rotation_and_translation=True 时会平移质心并用 Rotation.align_vectors 对齐结构。 晶体/slab 的 cell orientation 和 substrate reference frame 不应被任意旋转。
没有 cell path 或 variable-cell 支持 路径变量只包含 atomic positions 和可选旋转角。 相变、应变辅助迁移、可变晶胞 NEB 不适用。
并行模型不是 Slurm/MPI first-class PyDMF 并行是 Python threads 调 calculator。 ABACUS nested MPI、V100 分配和 image directories 需要 ATST wrapper 额外治理。

5.3 对典型 ATST 周期案例的判断

案例类型 是否可直接使用 PyDMF 建议
孤立分子反应、小分子重排、cluster 催化模型 较适合 可以优先作为 ATST DMF smoke/reference 案例，接 EMT/DP 或小规模 ABACUS。
固定 slab 上吸附/解离，adsorbate 不跨 cell 边界 可实验性尝试 关闭旋转/平动消除，固定 slab 原子或使用 constraints，并要求端点处于同一 cell 图像。
Li/Na 离子迁移，端点可能跨周期边界 不建议直接宣称支持 必须先做 unwrap/映射，最好在 fractional coordinate 或 MIC-aware displacement 中构造路径。
晶胞变化、相变、应变耦合路径 当前不适用 需要 cell degrees of freedom 和 metric-aware path formulation，超出 PyDMF 当前实现。
复杂表面反应伴随 slab 大重构 高风险实验性 可作为 TS guess 工具，但必须用 NEB/TS refinement/IRC 复核，且 FB-ENM 参数需人工审查。

6. ATST-Tools 加入 DMF 的可行性

6.1 有利条件

  • ATST-Tools 已以 ASE 为核心抽象，PyDMF 也以 ASE calculators 为后端接口，二者技术边界一致。
  • ATST 已具备 calculator factory、ABACUS/DP backend、per-image directory、endpoint single-point governance、artifact manifest、trajectory summary 和 TS validation 等通用能力。
  • D2S 工作流已经形成“double-ended rough path + single-ended refinement”的架构；在 DMF 的 PBC 语义和生产级案例证据稳定后，可插入为 rough path / TS guess generator。
  • DMF 对昂贵后端可能减少 evaluation points，适合 ABACUS/DFT 或 MLP 预筛选。

6.2 主要成本

  • 依赖成本：cyipopt 和 IPOPT 安装复杂，不能成为 ATST 默认依赖。应使用 optional extra 或运行时友好报错。
  • 并行成本：PyDMF thread parallel 与 SAI/Slurm/MPI resource model 不完全匹配，需要 ATST 包装 calculator directory 和资源声明。
  • 语义成本：DMF 输出的是 B-spline 路径和 tmax 历史，不是 NEB band；ATST 需要定义新的 artifact manifest 语义。
  • PBC 成本：周期体系需要 endpoint mapping、unwrap、MIC-aware distance、关闭刚体旋转和平动、constraints 传播等额外设计。

6.3 推荐集成路线

  1. P0: 文档和实验分支。保留本报告，明确 DMF 暂不在 feature matrix 标记为 supported。
  2. P1: 非周期基线。新增可选 standalone workflow，例如 calculation.type: dmf，先只承诺非周期小分子/cluster smoke；验证 PyDMF import guard、EMT/DP 后端、tmax 输出、trajectory 和 artifact manifest。
  3. P2: PBC 实验层。在 standalone DMF 中验证固定晶胞周期体系，支持 pbc_mode: cartesian_unwrapped 或等价显式端点映射策略，强制用户确认同一周期图像，默认关闭 rotation/translation removal，并记录 MIC/unwrap 限制。
  4. P3: 生产级案例验收。用 Li migration、H2/Au adsorbate dissociation、Cy/Pt 或 Zn migration 类案例做 ABACUS/DP 与 CI-NEB 对照；DMF TS guess + refinement 必须能给出可比 barrier、合理 reaction mode、endpoint connection 和 walltime 证据。
  5. P4: D2S 支持。在 P2/P3 证明 PBC 语义和生产案例可靠后，再把 DMF 接入 d2s.rough_method: dmf 或等价接口，复用 Dimer/Sella/CCQN、vibration 和 IRC validation。

7. 是否应支持到 ATST-Tools

建议支持，但采用分级、可选、实验性路线。DMF 与 ATST 的目标高度相关：ATST 已经是 ASE-native transition-state workflow toolkit，DMF 正好是 ASE calculator 生态中的新型 double-ended TS guess/path optimizer。将 DMF 纳入调研和实验 workflow 有明确价值。

但当前不应把 DMF 标记为与 NEB/AutoNEB 同等级的生产级周期体系方法，原因有三点：

  1. 官方论文口径和源码实现都没有严格周期体系支持。
  2. ATST 现有核心用户案例大量是 ABACUS/DP 周期体系，包括 Li 迁移、surface reaction 和 slab catalysis。
  3. DMF 最高能点仍是 TS candidate，必须纳入 ATST 已有 refinement/validation 链，而不是单独作为最终答案。

因此，最稳妥的产品定位是：DMF as an optional TS-guess accelerator for double-ended workflows。推进顺序应先证明非周期 API 和 PBC 语义，再用生产级案例决定是否进入 D2S；在文档和 CLI 中应明确“experimental”“non-periodic first”“PBC requires pre-unwrapped endpoints and validation”。

8. 推荐验收清单

阶段 验收内容 最低证据
P1 非周期 PyDMF import guard、EMT sample、trajectory/tmax/artifact 输出。 单元测试 + lightweight example。
P2 PBC experimental 固定 cell、同一周期图像、关闭 rotation/translation removal。 DP calculator 快速验证，路径不跨错误镜像。
P3 production candidate Li migration、H2/Au 或 Cy/Pt 类案例与 CI-NEB 对照。 barrier、TS mode、IRC/endpoint connection 和 walltime 记录。
P4 D2S integration DMF 输出 TS guess 可接 Dimer/Sella/CCQN，并作为 D2S rough stage 候选。 与现有 D2S smoke 和生产级 PBC 证据对齐，artifact manifest 完整。
