# ATST-Tools CLI and Restart Review

**Date**: 2026-05-10

## CLI Decision

ATST-Tools now uses a single git-style command, `atst`. The old standalone console commands are removed from packaging:

- `atst run config.yaml` replaces the former workflow runner.
- `atst neb make ...` replaces the former NEB path generator.
- `atst neb post ...` replaces the former NEB post-processor.

This is acceptable during the refactoring period because no stable 2.0 CLI compatibility contract has been declared.

## Lightweight Command Assessment

| Workflow | Lightweight CLI? | Decision |
| :--- | :--- | :--- |
| NEB / AutoNEB | Yes | Keep `atst neb make` and `atst neb post` for interpolation, barrier analysis, TS extraction, and vibration-index suggestions. |
| Dimer | Yes | Add `atst dimer make-from-neb` to generate `dimer_init.traj` and `displacement_vector.npy` from a completed NEB trajectory. |
| Sella | Not now | Sella can consume a TS structure from `atst neb post`; IRC should be designed later as a separate workflow. |
| D2S | No | D2S is inherently multi-stage and calculator-bound, so it remains YAML-driven through `atst run`. |
| Relax | No | Relaxation is calculator-bound; restart support belongs in `atst run --restart`. |
| Vibration | Yes | Add `atst vibration post` to rebuild summary and thermochemistry from existing ASE vibration cache without rerunning forces. |

## Restart Policy

- `atst run --restart config.yaml` is the CLI-level restart switch and overrides `calculation.restart` at runtime.
- NEB restarts from the latest band in the trajectory when available.
- AutoNEB preserves existing `run_autoneb*.traj` and `AutoNEB_iter/` when restart is enabled.
- Dimer and Sella restart from the last frame of their trajectory.
- D2S skips completed endpoint optimization, rough NEB, and single-ended stages when their checkpoint trajectories exist.
- Relax restarts from the last frame of the relaxation trajectory.
- Vibration restart keeps ASE cache files; `atst vibration post` handles post-processing-only recovery.
